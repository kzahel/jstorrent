JSTorrent Pairing Intent

Click the link below to trigger the JSTorrent pairing intent:

Launch JSTorrent Intent
Test Procedure:
  1. Click the blue link above
  2. Observe: Does it show the picker?
  3. Check the "Always" checkbox (if present)
  4. Select JSTorrent
  5. Click the link again
  6. Does the picker appear, or does it go straight to the app?
What to watch for:
• Does the picker say it's from this web page (instead of chrome-extension://)?
• Does checking "Always" persist for subsequent clicks?
• Is the behavior different from extension-triggered intents?

How to Test

Option 1: Open Directly (file://)

Open this file directly in Chrome. The source will be file://

Option 2: Use Python HTTP Server (http://)

# In the chromeos-testbed directory, run:
python3 -m http.server 8000

# Then open in Chrome:
# http://localhost:8000/intent-test.html

Option 3: Use HTTPS (if you have a server)

Host this on any HTTPS server and test from there.
